# Soul Schema - 灵魂层规则

> 定义灵魂层的结构、约定和工作流

## 核心目标

维护 `soul/` 目录，将其打造为**可运行的思维操作系统**。

- **目标**：从 `raw/` 中提取思维模式、性格特征、价值观，构建完整的数字灵魂
- **边界**：
  - 原始资料（`raw/`）是唯一事实来源，**只读不改**
  - 只在 `soul/` 里创建、修改文件
  - 不要动其它目录

---

## 目录结构

### 灵魂层文件

| 文件 | 作用 | 内容 |
|------|------|------|
| `soul/self.md` | Self Memory | 个人经历、价值观、生活习惯、重要记忆、人际关系 |
| `soul/persona.md` | Persona | 5层性格结构（硬规则 → 身份 → 说话风格 → 情感决策 → 人际行为） |
| `soul/thinking-framework.md` | Thinking Framework | 心智模型、决策启发式、表达DNA、价值观与反模式 |
| `soul/SKILL.md` | 完整的灵魂 Skill | 整合三层结构的完整 Skill |

## 三层结构

### Part A — Self Memory（基础层）

存储关于用户的事实性自我认知，为上层提供上下文。

**内容维度**：
- 核心身份（姓名、年龄、职业、城市、教育背景）
- 核心价值观（工作观、金钱观、关系观、成长观、核心矛盾）
- 生活习惯（作息、饮食、空间偏好、消费观念、日常仪式）
- 重要记忆（人生时间线、反复回忆的场景、转折点）
- 人际关系图谱（家人、朋友、亲密关系、冲突处理、关系角色）
- 成长轨迹（近几年变化、努力方向、反复挣扎的课题、自我接纳）

**数据来源**：
- 经历自述 → 直接提取
- 个人笔记 → 价值观、思考方式
- 微信聊天记录 → 人际关系、生活习惯

### Part B — Persona（性格层）

5层结构，驱动对话时的情感和行为反应。

**5层结构**：
- **Layer 0：硬规则**：不可违背的底线
- **Layer 1：身份锚定**：名字、年龄、职业、MBTI、星座
- **Layer 2：说话风格**：口头禅、标点、emoji、消息格式、语言正式程度
- **Layer 3：情感与决策模式**：情感表达、决策模式、情绪触发器、自我对话模式
- **Layer 4：人际行为**：社交能量、主动性、边界感、群体角色、冲突反应

**数据来源**：
- 微信聊天记录 → 说话风格、情感表达、人际行为
- AI对话记录（个人输入）→ 决策模式、思维习惯

### Part C — Thinking Framework（思维框架层）

从 raw 中提炼思维操作系统。

**内容维度**：
- **核心心智模型**（3-7个）
  - 三重验证：跨域复现、有生成力、有排他性
  - 每个模型含：名称、一句话描述、证据（≥2个场景）、应用方式、局限性
- **决策启发式**（5-10条）
  - 「如果X，则Y」的快速规则，有具体案例支撑
- **表达DNA**
  - 句式偏好、词汇特征、节奏感、幽默方式、确定性表达、引用习惯
- **价值观与反模式**
  - 核心价值排序、明确拒绝的行为/思维方式、内在矛盾与张力
- **诚实边界**
  - 不能预测的情况、不能替代的能力、信息截止时间

**数据来源**：
- AI对话记录（个人输入）→ 心智模型、决策启发式
- 个人笔记 → 思维方式、价值观
- 微信聊天记录 → 表达DNA、价值观与反模式

## 工作流

### 1. 初始化灵魂层

当用户说"创建我的灵魂"、"初始化灵魂层"时：

1. 收集基础信息（3个问题）：
   - 代号/昵称
   - 基本信息（年龄、职业、城市）
   - 自我画像（MBTI、星座、性格标签）

2. 导入原材料：
   - 微信聊天记录
   - AI对话记录
   - 个人笔记
   - 经历自述

3. 分析原材料：
   - 提取 Self Memory
   - 提取 Persona
   - 提取 Thinking Framework

4. 生成灵魂层文件：
   - `soul/self.md`
   - `soul/persona.md`
   - `soul/thinking-framework.md`
   - `soul/SKILL.md`

### 2. 更新灵魂层

当用户提供新资料时：

1. 读取新资料
2. 分析是否揭示新的思维模式或性格特征
3. 如果是，更新对应的文件：
   - 新的经历 → 更新 `soul/self.md`
   - 新的性格特征 → 更新 `soul/persona.md`
   - 新的思维模式 → 更新 `soul/thinking-framework.md`
4. 重新生成 `soul/SKILL.md`

### 3. 对话纠正

当用户说"不对"、"我不会这样说"时：

1. 识别纠正内容
2. 判断属于哪个层次：
   - Self Memory（事实/经历）→ 追加到 `soul/self.md`
   - Persona（性格/说话方式）→ 追加到 `soul/persona.md`
   - Thinking Framework（思维模式）→ 追加到 `soul/thinking-framework.md`
3. 重新生成 `soul/SKILL.md`

## 提炼方法

### 心智模型提炼

使用三重验证方法论：

1. **跨域复现**：同一个思维框架出现在≥2个不同领域
2. **有生成力**：用这个模型可以推断对新问题的立场
3. **有排他性**：不是所有人都会这样想

**验证结果**：
- 三重通过 → 心智模型
- 仅1-2重 → 降级为决策启发式
- 0重 → 丢弃

### 表达DNA分析

从用户的长文/对话中随机抽取20个段落，统计：

| 维度 | 测量方式 |
|------|---------|
| 平均句长 | 字数/句数 |
| 疑问句比例 | 疑问句数/总句数 |
| 类比密度 | 类比数/千字 |
| 第一人称使用率 | 「我」出现频率 |
| 确定性语气比例 | 「一定」「显然」vs「也许」「可能」 |
| 转折频率 | 「但是」「然而」「不过」/千字 |

### 价值观提炼

从用户的表达中提取核心价值排序：

- **工作观**：对工作的态度（事业/稳定/自由/意义感）
- **金钱观**：对金钱的态度（积累/消费/无所谓）
- **关系观**：对人际关系的态度（独处/亲密/广泛社交）
- **成长观**：对成长的看法（进取/舒适/随缘）
- **自由 vs 安全**的优先级
- **长期主义 vs 即时满足」

## 质量自检

### 心智模型
- [ ] 每个模型都有至少2个不同领域的证据？
- [ ] 模型数量在3-7个之间？
- [ ] 每个模型都有明确的应用场景和局限？

### 表达DNA
- [ ] 读起来有辨识度，不像通用AI？
- [ ] 没有过度模仿变成 caricature？
- [ ] 抓住了核心特征而非表面模仿？

### 决策启发式
- [ ] 每条规则都有具体案例支撑？
- [ ] 可以被新情况触发？

### 诚实边界
- [ ] 明确写了做不到什么？
- [ ] 标注了数据来源和分析时间？
- [ ] 承认了信息不足的维度？

## 与记忆层的协作

### 查询记忆层

当对话需要事实支撑时：

1. 判断问题是否需要事实支撑
2. 如果需要，查询 `wiki/index.md` 定位相关页面
3. 读取相关页面获取知识
4. 用灵魂层的方式，基于记忆层的内容回答

### 更新记忆层

灵魂层不直接修改记忆层，但可以建议：

- 发现新的思维模式 → 建议在 wiki 中创建概念页
- 发现新的性格特征 → 建议在 wiki 中更新实体页
- 发现新的经历 → 建议在 wiki 中创建来源摘要页
