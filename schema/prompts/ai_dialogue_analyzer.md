# AI 对话记录分析器

## 任务

从用户的 AI 对话记录中提取思维方式、决策模式和心智模型雏形。

**核心原则**：AI 对话记录是用户「认真思考」时的表达，比日常聊天更能反映深层思维模式。

---

## 提取维度

### 1. 问题分析方式

观察用户如何拆解和分析问题：

- **提问模式**：用户如何提出问题？
  - 是直接问答案，还是先描述背景？
  - 是问"怎么做"，还是问"为什么"？
  - 是问具体方案，还是问思考框架？

- **拆解方式**：用户如何拆解复杂问题？
  - 是按时间线拆解？
  - 是按逻辑关系拆解？
  - 是按优先级拆解？
  - 还是按其他维度？

- **关注点**：用户在分析问题时最关注什么？
  - 关注可行性？
  - 关注效率？
  - 关注风险？
  - 关注创新？
  - 关注用户体验？

```
输出格式：
提问模式：[描述 + 典型示例]
拆解方式：[描述 + 典型示例]
关注重点：[描述 + 典型示例]
```

### 2. 决策模式

观察用户如何做决策和权衡：

- **权衡方式**：用户如何权衡不同选项？
  - 列出 pros/cons？
  - 用权重打分？
  - 用直觉判断？
  - 寻找第三种方案？

- **风险态度**：用户如何对待风险？
  - 倾向保守还是激进？
  - 如何评估风险？
  - 如何应对不确定性？

- **取舍原则**：用户在做取舍时遵循什么原则？
  - 优先考虑什么？
  - 可以牺牲什么？
  - 绝对不能妥协的是什么？

```
输出格式：
权衡方式：[描述 + 典型示例]
风险态度：[描述 + 典型示例]
取舍原则：[描述 + 典型示例]
```

### 3. 思维框架雏形

从用户的提问和讨论中识别反复出现的思维模式：

- **高频概念**：用户反复使用的概念或术语
- **类比来源**：用户常用什么领域的知识来类比
- **思维工具**：用户习惯用什么工具/框架来思考
- **认知偏好**：用户倾向于什么类型的解释？

```
输出格式：
高频概念：[列表 + 出现场景]
类比来源：[列表 + 典型类比]
思维工具：[列表 + 使用场景]
认知偏好：[描述]
```

### 4. 价值判断

从用户的讨论中提取价值取向：

- **效率 vs 质量**：用户如何权衡？
- **创新 vs 稳定**：用户更倾向哪边？
- **独立 vs 协作**：用户的工作偏好？
- **短期 vs 长期**：用户的时间视角？
- **完美 vs 完成**：用户对完成度的要求？

```
输出格式：
效率观：[描述]
创新观：[描述]
工作观：[描述]
时间观：[描述]
质量观：[描述]
```

### 5. 表达特征（认真思考时）

观察用户在认真思考时的语言特征：

- **句式**：长句还是短句？
- **逻辑词**：常用什么逻辑连接词？
- **举例**：喜欢用什么类型的例子？
- **确定性**：表达确定还是留有余地？

```
输出格式：
句式偏好：[描述]
逻辑词偏好：[列表]
举例风格：[描述]
确定性程度：[描述]
```

---

## 分析原则

1. **区分「认真思考」vs「随便问问」**
   - 重点关注长消息、深入讨论、多次追问的对话
   - 忽略简短的、功能性的提问（如"帮我写个代码"）

2. **区分「真实想法」vs「试探性提问」**
   - 用户可能为了探索而提出对立观点
   - 重点关注用户最终选择的立场

3. **识别「思维演化」**
   - 如果用户的观点在对话中发生变化，记录变化过程
   - 标注「早期观点」和「近期观点」

4. **保留「矛盾」**
   - 如果用户在不同对话中表现出矛盾，保留矛盾
   - 矛盾本身是有价值的信号

---

## 输出要求

- 语言：中文
- 每个维度都要有具体示例支撑
- 没有足够信息的维度：标注 `（对话记录不足）`
- 避免过度推断，只提取有充分证据的结论
- 输出结果用于后续的 Thinking Framework 构建
