# Thinking Framework 生成模板

## 任务

从用户的 AI 对话记录、个人笔记和经历自述中，提炼用户的思维操作系统。

**核心理念**：不是记录用户说过什么，而是提炼用户 HOW to think。

---

## 一、心智模型提炼（3-7个）

### 提炼方法

使用三重验证方法论（来自 nuwa_skill）：

#### 验证1: 跨域复现
同一个思维框架出现在用户讨论的至少2个不同领域。

**示例**：
- 用户在讨论「工作效率」时说「先做最重要的事」
- 用户在讨论「学习方法」时说「先学最核心的概念」
- 用户在讨论「人际关系」时说「先维护最重要的关系」
→ 跨3个领域复现 → 这是真正的心智模型：「优先级思维」

#### 验证2: 有生成力
用这个模型可以推断用户对新问题的可能立场。

**示例**：
- 如果用户的核心心智模型是「第一性原理」
- 面对「如何选择职业」→ 推断用户会先问「什么是职业选择的本质？」
- 面对「如何投资」→ 推断用户会先问「投资的本质是什么？」
→ 能生成新推断 → 这是真正的心智模型

#### 验证3: 有排他性
不是所有人都会这样想，这个模型体现了用户的独特视角。

**示例**：
- 「反脆弱」是塔勒布的，不是所有人都这样看世界
- 「长期主义」也不是所有人都能做到
→ 有区分度 → 这是值得提炼的心智模型

### 提炼步骤

1. **扫描**：逐个读取 AI 对话记录、个人笔记、经历自述，列出所有候选论点
   - 用户反复表达的观点
   - 用户自创的术语或概念
   - 用户的核心主张
   - 通常会得到 15-30 个候选

2. **三重验证筛选**：对每个候选执行
   - 跨域复现：在≥2个不同领域/话题中出现？
   - 生成力：能推断对新问题的立场？
   - 排他性：不是所有人都这样想？
   - 三重通过 → 心智模型
   - 仅1-2重 → 降级为决策启发式
   - 0重 → 丢弃

3. **排序取舍**：按排他性强度排序（越独特越靠前），取top 3-7个
   - 宁少勿多——3个深刻的模型远好于10个浅薄的原则

4. **记录格式**：

```markdown
### 模型1: [名称]

**一句话**：[最简描述]

**证据**：
- 场景1：[用户在什么情况下表现出这个思维模式]
- 场景2：[另一个领域的表现]

**应用**：[遇到什么类型的问题时用这个镜片]

**局限**：[这个模型在什么情况下会失效]
```

## 二、决策启发式提炼（5-10条）

### 定义
用户做判断时的快速规则。可表述为「如果X，则Y」，有具体案例支撑。

### 提炼步骤

1. **识别决策场景**：从对话记录中找到用户做决策的场景
2. **提取规则**：观察用户的决策模式，提炼为规则
3. **验证普适性**：这个规则是否在多个场景中出现？
4. **记录案例**：每个规则至少有一个具体案例

### 记录格式

```markdown
1. **[规则名]**：[具体描述]
   - 应用场景：[什么时候用]
   - 案例：[已知的应用实例]
```

**示例**：
```markdown
1. **先做最重要的事**：面对多个任务时，先做最重要且不可推迟的
   - 应用场景：工作安排、学习计划、人生选择
   - 案例：用户在讨论时间管理时说「我会先做最重要且不可推迟的事，其他的可以后做或不做」
```

## 三、表达DNA分析

### 分析维度

| 维度 | 提取内容 | 数据来源 |
|------|---------|---------|
| 句式偏好 | 长句/短句、疑问/陈述、类比密度 | AI对话记录 |
| 词汇特征 | 高频词、专属术语、禁忌词 | 所有来源 |
| 节奏感 | 先结论还是先铺垫、转折方式 | AI对话记录 |
| 幽默方式 | 讽刺/自嘲/荒诞/冷幽默/不幽默 | 微信聊天记录 |
| 确定性表达 | 「我不确定」型 还是 「很明显」型 | AI对话记录 |
| 引用习惯 | 爱引谁、引什么类型 | 所有来源 |

### 量化方法

从用户的长文/对话中随机抽取20个段落，统计：

| 维度 | 测量方式 |
|------|---------|
| 平均句长 | 字数/句数 |
| 疑问句比例 | 疑问句数/总句数 |
| 类比密度 | 类比数/千字 |
| 第一人称使用率 | 「我」出现频率 |
| 确定性语气比例 | 「一定」「显然」vs「也许」「可能」 |
| 转折频率 | 「但是」「然而」「不过」/千字 |

### 风格标签

从以下维度打标：

```
正式 ←→ 口语
抽象 ←→ 具体
谨慎 ←→ 断言
学术 ←→ 通俗
长句 ←→ 短句
铺垫型 ←→ 结论先行
数据驱动 ←→ 叙事驱动
```

### 记录格式

```markdown
## 表达DNA

角色扮演时必须遵循的风格规则：
- 句式：[长句/短句偏好、疑问/陈述比例]
- 词汇：[高频词、专属术语、禁忌词]
- 节奏：[先结论还是先铺垫、转折方式]
- 幽默：[讽刺/自嘲/荒诞/冷幽默/不幽默]
- 确定性：[「我不确定」型 还是 「很明显」型]
- 引用习惯：[爱引谁、引什么类型]
```

## 四、价值观与反模式

### 价值观提炼

从用户的表达中提取核心价值排序：

- **工作观**：对工作的态度（事业/稳定/自由/意义感）
- **金钱观**：对金钱的态度（积累/消费/无所谓）
- **关系观**：对人际关系的态度（独处/亲密/广泛社交）
- **成长观**：对成长的看法（进取/舒适/随缘）
- **自由 vs 安全**的优先级
- **长期主义 vs 即时满足**

### 反模式提炼

用户明确反对的行为/思维方式：

- 用户批评过的行为
- 用户明确表示不会做的事
- 用户厌恶的思维模式

### 内在矛盾

价值观之间的内在冲突（深度的来源）：

- 既追求自由又重视纪律
- 既渴望稳定又厌倦平庸
- 既想独立又害怕孤独

**处理原则**：矛盾是人格的核心特征，不是需要修复的Bug。

### 记录格式

```markdown
## 价值观与反模式

**我追求的**：[排序的价值观]
**我拒绝的**：[明确的反模式]
**我自己也没想清楚的**：[内在矛盾和张力]
```

---

## 五、诚实边界

必须明确写出的局限：

- 不能预测面对全新问题的反应
- 不能替代用户的创造力和直觉
- 用户的真实想法可能比表达出来的更复杂
- 信息截止到分析时间点

---

## 质量自检清单

生成 Thinking Framework 后，用以下问题自检：

### 心智模型
- [ ] 每个模型都有至少2个不同领域的证据？
- [ ] 模型数量在3-7个之间？（太少=太浅，太多=没提炼）
- [ ] 每个模型都有明确的应用场景和局限？
- [ ] 模型之间有张力但不矛盾？

### 表达DNA
- [ ] 读起来有辨识度，不像通用AI？
- [ ] 没有过度模仿变成 caricature？
- [ ] 抓住了核心特征而非表面模仿？

### 决策启发式
- [ ] 每条规则都有具体案例支撑？
- [ ] 可以被新情况触发（不只适用于原始案例）？

### 诚实边界
- [ ] 明确写了做不到什么？
- [ ] 标注了信息源和分析时间？
- [ ] 承认了信息不足的维度？

### 整体
- [ ] 用用户的眼睛看一个新问题，能得到有价值的视角？
- [ ] 不是用户原话的拼凑，而是框架的运行？
- [ ] 删掉名字后，还能认出这是谁的思维方式？
